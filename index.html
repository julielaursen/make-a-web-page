Lesson 1 Notes: HTML and the WWW

The basics of the Web and HTML

HTML stands for hypertext markup language and is what glues everything on the web together. It makes the web web-like by using hyperlinks. It was invented in 1990s and has over 30 billion pages. Although the main type of document on the web is HTML, the web contains all kinds of files: plain text, HTML, images, videos and audio. The major pieces of the web are the user, the computer (which has a browser), the Internet, and the Server. The web works as follows: a user types the URL of a website into a browser, the computer then sends an HTTP request to a server, then the server finds the appropriate HTML documents and sends it back to the computer which then reads it and displays it in its browser to the user.

HTML documents are the heart of the web, and HTML is made up of text content (which is what you see) and markup content (which affects how it looks). The markup itself is written using tags. Opening tags and closing tags with content in between form HTML elements. The opening and closing tags have the same name but the closing tag has a backslash before it. Tags include the <b> (bold) tag, <em> (italic), <br> (line break), etc. Void tags have no closing tags because they have no content within them. Tags could also include attributes such as the href attribute inclosed in the “anchor” tag <a href=”URL”>. Src and alt are also attributes. Computers are Stupid and Thinking Like a Programmer

Remember computers take instructions very literally. They're sensitive to small types and mistakes and often programmers have to "debug" to track and correct their mistakes.

Technological empathy involves five concepts:
Procedural Thinking. Procedural thinking involves creating perfectly clear and unambiguous instructions for a computer to follow (because remember, they're stupid)
Abstract Thinking Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things and avoid unnecessary repetition. Unnecessary repitition is important because when programmers don't avoid it, they can type or copy paste the same code (often with minor errors) many times, and then when they have to change something, easily miss changing all instances, leading to nasty errors!
Systems Thinking This happens hwen you break a big problem into small er pieces.
Technological empathy This is the ability to understand how computers work and what it's good and bad at doing
Debugging Debugging is a systematic process of relentlessly identifying the cause of a computer program that doesn't work.

Whitespace, Inline, Block and Container Elements

The <br> tag represents “inline” formatting where the markup signifies end of the a line, while the <p> (paragraph) tag represents the “block” formatting, where the markup signifies an invisible box around the text. Span and div are container elements. Span is inline and div is block. These elements only contain content and styles can be attached to them to change the way they display that content.

HTML and CSS

HTML and CSS are both considered "languages." HTML is the language of the web and has syntax and rules. HTML controls the structure of a webpage and CSS controls the style of a webpage. HTML provides the browser with info on what style (CSS) to use and where to get it from.

DOM and Tree Structures

When programmers talk about the DOM (domain object model), they are talking about the tree-like structure of a page. A tree diagram is a way of representing the hierarchical nature of a structure in graphical form. Tree elements are called "nodes" and lines connecting elements are called "branches."

Structure is important here because the more indented an element is, the more deeply nested its corresponding "box" is. This makes it easier to skim the "tree" and see which elements are the "parents" and "children" of eachother. A good reference on branches: Tree_structures

The Box model in HTML/CSS

The box model in HTML consists of margin, border, padding and content in that order with content in the center of the box. To style the box size in CSS use box-sizing: border-box and this will include the border and the padding along with the size of the box for ease of use. You can set boxes in size as either a set number of pixels or as percentages. If you set it to pixels, the browser will take that exact size and will not change responsively with the screen. If you set the box to a percentage, it will change along with the screen.

Box Positioning in HTML/CSS

Divs are block elements and not inline, so by default they take up the entire width of a page. Adding the display: flex will override this behavior and let divs appear next to eachother.
